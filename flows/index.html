RNA-seq flow family

RNA-seq 流程家族

The TAFFISH RNA-seq suite is organized as a family of small, versioned, auditable flow apps. Each subflow has its own stable biological responsibility, test data route, command provenance, logs, and output contract. rnaseq-standard-flow is the umbrella command that connects those subflows into one end-to-end run; it does not hide a second analysis implementation behind the scenes.

TAFFISH RNA-seq 套件由一组小而清楚、可版本化、可审计的流程应用组成。每个子流程都有稳定的生物学职责、测试数据路线、命令溯源、日志和输出契约。rnaseq-standard-flow 是把这些子流程串起来的一站式总入口；它不是在内部重新写一套隐藏分析逻辑，而是显式组合已经发布和验证的子流程。

How the flows connect

这些流程如何连接

The first lane is the default Salmon-first expression route. The second lane is optional and adds genome-aware alignment, BAM-level QC, and featureCounts evidence when users need alignment evidence in addition to transcript quantification.

The standard flow is an umbrella, not a black box

standard-flow 是总入口，不是黑盒

rnaseq-standard-flow is designed for users who want to start from ordinary local FASTQ files, a genome FASTA, a matching annotation file, sample metadata, and optional offline gene sets, then receive a coherent analysis directory and a bilingual HTML report. Its default behavior keeps the analysis lightweight: it builds a reference package, quantifies reads with Salmon, imports gene-level counts through tximport, runs differential expression with DESeq2, runs enrichment when gene sets are provided, and finally collects all major outputs into a report.

rnaseq-standard-flow 面向希望从普通本地 FASTQ、基因组 FASTA、配套注释、样本元数据和可选离线基因集直接开始分析的用户。默认行为保持轻量和实用：先构建参考包，再用 Salmon 完成表达定量，通过 tximport 汇总到基因层面，使用 DESeq2 做差异表达，提供基因集时继续做富集分析，最后把主要结果、图、表、QC 子报告、日志和溯源统一收集成双语 HTML 报告。

When --route both is selected, the umbrella also asks the index flow to build a HISAT2 genome index, runs genome-aware alignment, produces sorted BAM files, derives featureCounts matrices, and runs alignment-level QC. Those outputs do not replace the default expression route unless the user explicitly asks DESeq2 to use featureCounts with --de-source featurecounts. This preserves a fast default while still giving advanced users a genome-evidence branch.

当选择 --route both 时，总流程还会要求参考流程构建 HISAT2 基因组索引，继续运行基因组比对，生成排序 BAM，基于 BAM 产生 featureCounts 矩阵，并执行比对层面 QC。这些输出默认不会替代表达定量路线；只有用户显式指定 --de-source featurecounts 时，DESeq2 才会改用 featureCounts 计数。这样既保留快速默认路线，也给高级用户提供可审查的基因组证据分支。

Flow responsibilities

每个流程的职责

rnaseq-index-flow

0.1.0-r1

This flow turns a genome and annotation release into the reusable reference contract consumed by downstream RNA-seq flows. It validates sequence IDs, standardizes the annotation, produces GTF/GFF3-derived files, extracts transcript FASTA, writes tx2gene.tsv, and builds Salmon and Kallisto transcriptome indexes. When requested, it also builds a HISAT2 genome index so alignment-based evidence can join the same analysis family.

这个流程把基因组和注释版本整理成后续 RNA-seq 流程可复用的参考契约。它会检查 FASTA 与注释中的序列 ID 是否一致，标准化注释，生成下游需要的 GTF/GFF3 衍生文件，提取转录本 FASTA，写出 tx2gene.tsv，并构建 Salmon 和 Kallisto 的转录组索引。如果用户需要比对证据，它还可以额外构建 HISAT2 基因组索引，使表达定量和基因组比对共享同一套参考来源。

rnaseq-expression-flow

0.1.0-r1

This is the default abundance route. It reads the sample table, runs read-level QC and trimming, quantifies each sample against the Salmon index, collects quantification files, imports transcript evidence to gene-level matrices with tximport, and summarizes QC through MultiQC. Its biological job is not to call genes significant; it measures how much transcript-derived signal each sample carries so later statistical steps can compare groups.

这是默认表达丰度路线。它读取样本表，对测序 reads 做基础 QC 和修剪，将每个样本定量到 Salmon 索引，收集定量文件，再通过 tximport 把转录本层面的证据汇总成基因层面矩阵，并用 MultiQC 汇总 QC 信息。它的生物学职责不是判断基因是否显著变化，而是可靠地测量每个样本承载了多少表达信号，为后续分组比较提供矩阵基础。

rnaseq-de-flow

0.1.0-r2

This flow receives a gene count matrix and sample metadata, then performs DESeq2-based differential expression under an explicit design and contrast. It produces normalized counts, statistical result tables, PCA, sample correlation, MA and volcano plots, heatmaps, differential gene count summaries, expression distributions, and top-gene visualizations. In the route, it is the point where measured expression becomes a statistical statement about biological conditions.

这个流程接收基因计数矩阵和样本元数据，并在显式设计公式与比较组设定下运行 DESeq2 差异表达分析。它输出归一化计数、统计结果表、PCA、样本相关性、MA 图、火山图、热图、差异基因数量汇总、表达分布图和 top gene 可视化。在整条路线中，这是表达测量转化为“不同生物条件之间是否存在系统性变化”的统计判断节点。

rnaseq-enrichment-flow

0.1.0-r3

This flow interprets differential expression at the gene-set level. It uses offline GMT resources and an optional background list to run ORA for significant gene sets and GSEA for ranked genes, then generates coordinated dotplots, barplots, upset-style summaries, NES plots, and leading-edge style outputs. It helps turn a list of genes into biological hypotheses about pathways, processes, or compartments, while keeping the input universe explicit.

这个流程把差异表达结果提升到基因集层面解释。它使用离线 GMT 资源和可选背景基因列表，对显著基因列表进行 ORA，对完整排序基因进行 GSEA，并生成风格统一的气泡图、柱状图、集合关系摘要、NES 图和 leading-edge 类结果。它的意义是把一串差异基因转化成关于通路、过程或细胞组分的生物学假设，同时保留被检验基因背景的透明性。

rnaseq-alignment-flow

0.1.0-r1

This optional flow maps reads back to the genome with HISAT2 and produces coordinate-sorted BAM files plus indexes. It is useful when a project needs genome-aware evidence, browser inspection, splice-aware alignment context, or a BAM substrate for featureCounts and alignment QC. It is not the default quantification engine because alignment costs more time and storage, but it is valuable when users need an independent evidence layer.

这是可选的基因组比对流程，使用 HISAT2 将测序 reads 比对回基因组，并生成坐标排序 BAM 及其索引。当项目需要基因组层面的证据、浏览器查看、剪接感知比对背景，或需要给 featureCounts 和比对质控提供 BAM 底物时，它非常有价值。它不是默认表达定量引擎，因为比对会增加时间和存储成本；但在需要独立证据层时，它能补足 Salmon-first 路线。

rnaseq-count-flow

0.1.0-r1

This flow converts aligned BAM evidence into a featureCounts gene-level count matrix. It sits after alignment and before optional DE from featureCounts. The matrix it produces is especially useful for users who prefer annotation-overlap counting, need comparability with older alignment-first pipelines, or want to compare Salmon/tximport counts against a genome-alignment-derived count source.

这个流程把已经比对好的 BAM 证据转换成 featureCounts 基因层面计数矩阵。它位于比对流程之后，也可以作为 featureCounts 来源差异分析的上游。它的输出适合偏好注释区间重叠计数的用户，也适合需要和传统“先比对”流程对齐，或希望把 Salmon/tximport 计数与基于基因组比对的计数进行交叉检查的项目。

rnaseq-alignment-qc-flow

0.1.0-r1

This optional QC flow evaluates BAM-level evidence with SAMtools, RSeQC, Qualimap, and MultiQC. It asks whether reads map consistently, whether libraries show expected RNA-seq behavior, whether coverage and assignment patterns look plausible, and whether any sample behaves like an outlier. It does not create expression matrices; it explains whether the alignment branch is technically trustworthy.

这是可选的 BAM 层面 QC 流程，结合 SAMtools、RSeQC、Qualimap 和 MultiQC 评估比对证据。它关心测序 reads 是否稳定比对、文库是否呈现 RNA-seq 应有特征、覆盖和分配模式是否合理、是否存在异常样本。它不会生成表达矩阵；它的职责是说明比对分支在技术上是否可信，帮助用户判断后续基于 BAM 的解释是否稳固。

rnaseq-report-flow

0.1.0-r4

This flow is the delivery and interpretation layer. It collects outputs from upstream modules, separates plots and tables, links QC HTML reports, writes project summaries, embeds provenance, and produces a bilingual HTML report plus a longer interpretation guide. It intentionally does not rerun analysis; its value is to organize results by biological question so wet-lab users, bioinformaticians, and non-specialist readers can navigate the project.

这是交付和解读层。它收集上游模块输出，整理图和表，链接 QC HTML 子报告，写出项目摘要，嵌入命令、版本和 manifest 溯源，并生成双语 HTML 主报告与更长的解读指南。它有意不重新运行分析；它的价值是按生物学问题组织结果，使湿实验用户、生信用户和非专业读者都能理解项目发生了什么、应该先看哪里、哪些结论需要谨慎解释。

rnaseq-standard-flow

0.1.0-r2

This is the end-to-end public entrypoint. It composes the index, expression, DE, enrichment, report, and optional alignment/count/QC subflows while preserving their individual output directories and provenance. For most users it is the easiest command to run on real project data; for maintainers it remains inspectable because each major biological step is delegated to a named TAFFISH flow with its own tests and release history.

这是最终面向用户的一站式入口。它组合参考构建、表达定量、差异表达、功能富集、报告生成以及可选的比对、计数、质控子流程，同时保留各子流程自己的输出目录和溯源信息。对大多数用户来说，它是直接分析真实项目数据的最方便命令；对维护者来说，它仍然保持可审计性，因为每个主要生物学步骤都委托给一个具名 TAFFISH 流程，并拥有自己的测试与发布历史。

How to choose a route

如何选择路线

Use the default route when

这些情况使用默认路线

  • the goal is expression matrix, DESeq2, enrichment, and final report;
  • 目标是表达矩阵、DESeq2、富集分析和最终报告；
  • the project does not require BAM inspection or alignment-level QC;
  • 项目不需要检查 BAM 或做比对层面 QC；
  • speed, simplicity, and a compact output directory matter.
  • 速度、简单性和较小输出目录更重要。

Add --route both when

这些情况添加 --route both

  • you need sorted BAM files for review, archival, or external tools;
  • 需要排序 BAM 用于人工检查、归档或外部工具；
  • you want featureCounts matrices as an alignment-derived comparison;
  • 希望得到基于比对的 featureCounts 矩阵作为对照；
  • you want RSeQC/Qualimap/MultiQC alignment evidence in the final report.
  • 希望最终报告中包含 RSeQC、Qualimap、MultiQC 等比对证据。

Use --de-source featurecounts only when the alignment/count route is enabled and the scientific decision is to base DESeq2 on featureCounts rather than Salmon/tximport counts.

只有在启用了 alignment/count 路线，并且科学上决定让 DESeq2 使用 featureCounts 而不是 Salmon/tximport counts 时，才需要 --de-source featurecounts。
