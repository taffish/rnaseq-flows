rnaseq-report-flow 0.1.0-r4 collects existing RNA-seq flow outputs into an enhanced static project report.

It also writes an offline companion interpretation guide for report reading order, plot roles, and common interpretation boundaries.

It does not rerun expression quantification, alignment, counting, QC, differential expression, or enrichment. Biological interpretation remains the user responsibility.

## Upstream module: standard

rnaseq-standard-flow 0.1.0-r2 runs the r2 RNA-seq standard route by composing published TAFFISH RNA-seq subflows.

The default route prepares a reference bundle, quantifies transcript expression with Salmon/tximport, runs DESeq2 differential expression, runs offline GMT enrichment, and renders a static project report.
With --route both it also runs HISAT2 alignment, alignment-level QC, and featureCounts gene counting. Differential expression uses --de-source to choose Salmon/tximport or featureCounts counts.
The flow does not download references, gene sets, or databases at runtime. It writes all generated files under the explicit output directory.


## Upstream module: reference

rnaseq-index-flow prepared a reference bundle for TAFFISH RNA-seq flows.
The run used mode=genome_annotation, indexer=salmon, and genome_indexer=hisat2. The genome FASTA and annotation were supplied by the user; AGAT normalized the annotation, gffread wrote GTF and transcript FASTA, and a strict tx2gene.tsv table was derived from transcript features.
A HISAT2 genome index was built from the same genome FASTA for downstream splice-aware alignment.
The output directory contains recorded commands, tool versions, logs, a manifest, summary tables, and the requested transcriptome and genome index files.


## Upstream module: expression

rnaseq-expression-flow quantified RNA-seq FASTQ samples with Salmon and summarized transcript abundances with tximport.
The run used Salmon library type A and countsFromAbundance=no. Reads were quantified directly without fastp trimming.
FastQC was run on the input FASTQ files.
The output directory contains Salmon quant.sf files, gene-level matrices, transcript-level matrices, MultiQC report files, logs, command provenance, tool versions, and a machine-readable manifest.


## Upstream module: alignment

rnaseq-alignment-flow 0.1.0-r1 aligned RNA-seq FASTQ samples to a user-supplied HISAT2 genome index and produced coordinate-sorted BAM files. Reads were aligned without fastp preprocessing. SAM files were sorted and indexed with SAMtools; per-sample flagstat and idxstats summaries were written for downstream QC. The flow records commands, dependency versions, methods, logs, BAM paths, and a machine-readable manifest. r1 supports HISAT2; STAR, featureCounts, and RSeQC/Qualimap are intentionally left to later or downstream flows.


## Upstream module: count

rnaseq-count-flow 0.1.0-r1 counted reads from coordinate-sorted BAM files with featureCounts and produced a gene-level count matrix.
The run used strand=0, paired=false, feature_type=exon, attribute=gene_id, and min_mapq=0. BAM inputs were validated with samtools quickcheck and were not modified. The featureCounts annotation was prepared as a local copy so missing target attributes could be filled from transcript metadata when needed. featureCounts raw output, assignment summaries, the downstream gene_counts.tsv matrix, MultiQC, step logs, commands, versions, methods, and a manifest were written under the output directory.


## Upstream module: alignment_qc

rnaseq-alignment-qc-flow 0.1.0-r1 evaluated coordinate-sorted RNA-seq BAM files with SAMtools, RSeQC, Qualimap, and MultiQC.
The run used mapq=30, infer_sample_size=200000, sequencing_protocol=non-strand-specific, paired=false, threads=2, and java_mem_size=4G. BAM files and indexes were validated and were not modified. The GTF was copied locally, and a BED gene model was either copied from --annotation-bed or generated locally with gffread for RSeQC. Per-sample SAMtools, RSeQC, Qualimap, MultiQC, summary, commands, versions, methods, and manifest files were written under the output directory.


## Upstream module: de

rnaseq-de-flow ran a DESeq2 differential expression analysis from a gene count matrix and sample metadata.
The design formula was ~ condition, and the primary contrast was snf2_KO versus WT for factor condition. Genes were reported as significant when adjusted p-value was at or below 0.05 and absolute log2 fold change >= 1. Log2 fold-change shrinkage was not requested; results_shrunken.tsv is a stable contract copy of results.tsv.
The output directory contains differential expression tables, normalized and VST matrices, unified PCA/MA/volcano/heatmap plots, sample correlation and expression distribution QC plots, DE count and top-gene expression plots, downstream gene lists, step logs, command provenance, tool versions, and a machine-readable manifest.


## Upstream module: enrichment

rnaseq-enrichment-flow 0.1.0-r3 ran offline GMT-based enrichment analysis. ORA used 354 query genes against an explicit background of 7127 genes. Preranked GSEA used 5662 ranked genes. Gene sets were read from a user-supplied GMT file; gene identifiers were treated as exact strings with no ID conversion, organism inference, online lookup, or gene-set download. The primary dotplot was re-rendered by the flow with wrapped term labels for readability. The dotplot.original.pdf/png files are an optimized classic/original-style re-rendering with wider labels; the raw dependency plot remains in 02_intermediate/enrichment-r/dotplot.pdf/png for audit. The flow also rendered an ORA adjusted-p-value barplot, a GSEA normalized-enrichment-score plot, GSEA running enrichment score curves, and plot_summary.tsv. Parameters were min_size=2, max_size=500, pvalue_cutoff=1, padj_method=BH, top_n=20, seed=1.
