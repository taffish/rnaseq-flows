Interpretation结果解读

How to read the final report

如何阅读最终报告

Start with the overview and QC sections before interpreting biological signal. Low-quality reads, outlier samples, poor assembly quality, missing annotation resources, or inconsistent mapping evidence can change downstream conclusions.

解读生物学信号前，先看总览和 QC 部分。低质量 reads、异常样本、组装质量差、注释资源不足或比对证据不一致，都会影响后续结论。

Expression matrices

表达矩阵

In reference mode, gene counts are used for statistical testing and TPM-like values support inspection. In de novo mode, the default matrix is transcript-level unless a trusted tx2gene or cluster mapping was supplied.

有参模式中，gene counts 用于统计检验，TPM 类结果适合人工检查。无参模式中，默认矩阵是转录本层面，除非提供可信 tx2gene 或 cluster 映射。

De novo-specific sections

无参特有章节

Read assembly statistics, N50, filtered transcript count, BUSCO status, annotation coverage, and matrix semantics before comparing DE and enrichment. A strong de novo report should make clear which features were assembled, which were quantified, which were annotated, and which ID space was used for enrichment.

比较差异和富集之前，应先阅读组装统计、N50、过滤后转录本数量、BUSCO 状态、注释覆盖度和矩阵语义。好的无参报告需要清楚说明哪些特征被组装、哪些被定量、哪些获得注释，以及富集分析使用了哪个 ID 空间。

Differential expression

差异表达

Read PCA and correlation plots first, then MA/volcano plots, then feature tables and heatmaps. Effect size and adjusted P value answer different questions. In de novo mode, “significant features” may be transcripts rather than known genes.

先看 PCA 和相关性图，再看 MA/volcano 图，最后看 feature table 和 heatmap。效应量和校正 P 值回答的是不同问题。无参模式中，“显著特征”可能是转录本，而不是已知基因。

Enrichment

功能富集

ORA summarizes significant-feature lists, while GSEA reads the full ranked list. Reference enrichment depends on known gene-set resources; de novo enrichment depends on homology annotation and GO transfer. Treat enrichment as a biological hypothesis generator, not proof that a pathway is active.

ORA 汇总显著特征列表，GSEA 读取完整排序列表。有参富集依赖已知基因集资源；无参富集依赖同源注释和 GO 转移。富集结果更像生物学假设生成器，不应直接等同于通路被证明激活。
