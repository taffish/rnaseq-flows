rnaseq-standard-flow

The standard flow is the public umbrella command for a complete bulk RNA-seq route. It composes the stable subflows rather than reimplementing them: reference preparation, Salmon expression quantification, DESeq2 differential expression, GMT-based enrichment, optional HISAT2/featureCounts evidence, and final bilingual reporting.

rnaseq-standard-flow 是完整 bulk RNA-seq 路线的公开总入口。它不是重新实现所有步骤，而是组合已经稳定的子流程：参考准备、Salmon 表达定量、DESeq2 差异表达、基于 GMT 的富集分析、可选 HISAT2/featureCounts 证据分支，以及最终双语报告。

Minimal standard command

最小标准命令

taf-rnaseq-standard-flow \
  --samples samples.tsv \
  --genome genome.fa \
  --annotation genes.gff3 \
  --metadata metadata.tsv \
  --design '~ condition' \
  --contrast condition:treated:control \
  --gene-sets gene_sets.gmt \
  --background background.tsv \
  --outdir rnaseq-standard-out \
  --threads 8

Input files and how to build them

输入文件与构建方式

The standard flow starts from ordinary FASTQ files, a matching genome and annotation, a sample design table, and offline gene sets. It does not require pre-running other TAFFISH RNA-seq flows; the subflows are called internally.

标准流程可以直接从普通下机 FASTQ、匹配的基因组与注释、样本设计表和离线基因集开始运行。用户不需要预先手动运行其他 TAFFISH RNA-seq 子流程；这些子流程会在标准流程内部被调用。

samples.tsv

One row per biological sample. Required columns are sample_id and read1; paired-end samples also use read2. sample_id must be unique, should avoid spaces and shell metacharacters, and should match the metadata sample IDs. Relative FASTQ paths are resolved relative to the directory containing samples.tsv.

每个生物学样本一行。必需列是 sample_id 和 read1；双端样本还使用 read2。sample_id 必须唯一，建议避免空格和 shell 特殊字符，并且应与 metadata 中的样本 ID 一致。FASTQ 相对路径按 samples.tsv 所在目录解释。

metadata.tsv

One row per sample, with a sample ID column named sample by default. Other columns describe biological condition, batch, sex, time point, or other covariates. Variables used in --design and --contrast must appear here.

每个样本一行，默认样本 ID 列名为 sample。其他列用于描述生物学条件、批次、性别、时间点或其他协变量。--design 和 --contrast 中使用的变量必须来自这张表。

Reference FASTA and annotation

参考 FASTA 与注释

--genome and --annotation should come from the same genome release. Chromosome or contig IDs in the annotation must match the first token of the FASTA headers. Gene IDs produced from the annotation become the ID space for expression, DE, background, and enrichment.

--genome 和 --annotation 应来自同一个参考版本。注释中的染色体或 contig ID 必须匹配 FASTA header 的第一个字段。流程从注释中得到的 gene ID 会成为表达矩阵、差异分析、背景基因和富集分析的共同 ID 空间。

GMT gene sets and background

GMT 基因集与背景

--gene-sets is a local GMT file. Each line is set ID, description, then genes. --background is recommended for ORA and should contain the genes that could have been detected or tested in this experiment.

--gene-sets 是本地 GMT 文件。每行依次是集合 ID、描述、多个基因。--background 推荐提供给 ORA，内容应是本实验中理论上可检测或实际参与检验的基因集合。

For differential expression, the usual simple model is --design '~ condition' and --contrast condition:treated:control. If the experiment has known batches, use a model such as --design '~ batch + condition' so the comparison of interest is still the condition effect after accounting for batch.

差异表达最常见的简单模型是 --design '~ condition' 与 --contrast condition:treated:control。如果实验存在已知批次，建议使用类似 --design '~ batch + condition' 的模型，使主要比较仍然是在控制批次后的 condition 效应。

Route choices

路线选择

The default route is Salmon-first and is suitable for expression matrices, DESeq2, enrichment, and reporting. Use --route both when you also need sorted BAM, featureCounts, and alignment QC evidence. Add --de-source featurecounts only when DESeq2 should use featureCounts counts instead of Salmon/tximport counts.

默认路线是 Salmon-first，适合表达矩阵、DESeq2、富集分析和报告。需要排序 BAM、featureCounts 和比对质控证据时使用 --route both。只有当你希望 DESeq2 使用 featureCounts 而不是 Salmon/tximport 计数时，才添加 --de-source featurecounts。

Default expression route: reference preparation -> Salmon quantification -> tximport gene matrix -> DESeq2 -> enrichment -> report. Evidence route: the same route plus HISAT2 alignment, alignment QC, and featureCounts. The second route is heavier but gives genome-aware BAM evidence and an independent gene-count matrix.

默认表达路线：参考准备 -> Salmon 定量 -> tximport 基因矩阵 -> DESeq2 -> 富集分析 -> 报告。证据分支：在同一主线之外增加 HISAT2 比对、比对质控和 featureCounts。第二条路线更重，但会提供基因组位置层面的 BAM 证据和一套独立的 gene-count 矩阵。

--samplesyes是noneFASTQ sample table for expression and optional alignment. It must contain sample_id and read1; add read2 for paired-end data. --outdir, -oyes是noneDedicated output directory. Existing directories are refused unless --force is used.专用输出目录。目录已存在时默认拒绝运行，除非使用 --force。

Boundaries

边界

The flow does not download reference resources during normal execution, does not replace experimental design review, and does not make final biological conclusions automatically. For expression-only projects without contrasts, use the expression outputs and skip biological interpretation of DE/enrichment sections.

流程正常运行时不下载参考资源，不替代实验设计审查，也不会自动给出最终生物学结论。只有表达矩阵需求、没有可比较分组时，可以使用 expression 输出，并跳过对差异和富集部分的生物学解读。
