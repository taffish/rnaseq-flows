FAQ常见问题

Common boundary questions

常见边界问题

Can I run only expression quantification?

只有表达矩阵需求可以运行吗？

Yes. Use the expression flow, the de novo expression flow, or run standard-flow and use the expression outputs. Differential expression requires meaningful experimental groups.

可以。可以单独使用有参 expression flow、无参 expression flow，或运行 standard-flow 后只使用表达矩阵输出。差异表达需要有意义的实验分组。

Is the workflow reference-only?

流程只能做有参吗？

No. The default is reference mode, but rnaseq-standard-flow 0.3.0-r1 also supports explicit --mode denovo for no-reference analysis.

不是。默认是有参模式，但 rnaseq-standard-flow 0.3.0-r1 也支持显式 --mode denovo 进行无参分析。

What are protein-db and GO map?

protein-db 和 GO map 是什么？

They are external annotation resources for de novo analysis, not sequencer outputs. The protein FASTA supports homology annotation; the GO map allows annotation-derived enrichment.

它们是无参分析用的外部注释资源，不是测序下机文件。蛋白 FASTA 支持同源注释；GO map 支持注释派生富集。

Does de novo mode produce gene-level results?

无参模式会产生基因层面结果吗？

By default, no. The safe default is transcript-level. Gene or cluster-level matrices require an explicit tx2gene or cluster mapping.

默认不会。安全默认是转录本层面。只有显式提供 tx2gene 或 cluster mapping 时，才生成 gene 或 cluster 层面矩阵。

Does the flow download databases?

流程会下载数据库吗？

No. Genome, annotation, gene sets, background lists, protein databases, GO maps, and BUSCO lineage data are explicit inputs.

不会。基因组、注释、基因集、背景列表、蛋白数据库、GO map 和 BUSCO lineage 都是显式输入。

Why can BUSCO be skipped?

为什么 BUSCO 可能被跳过？

BUSCO needs a lineage dataset. If no local lineage is supplied, de novo assembly still runs and the report records BUSCO as skipped rather than downloading data.

BUSCO 需要 lineage 数据集。如果未提供本地 lineage，无参组装仍会运行，报告会记录 BUSCO 为 skipped，而不会联网下载数据。

Is this a Nextflow replacement?

这是 Nextflow 替代品吗？

No. TAFFISH standardizes versioned command units and shell-native routes. Workflow engines can still orchestrate larger production DAGs.

不是。TAFFISH 标准化可版本化命令单元和 shell-native 路线。大型生产 DAG 仍然可以由工作流引擎编排。

Why separate report-flow?

为什么报告单独做成一个流程？

Reporting is a static collection and interpretation layer. It should aggregate upstream outputs rather than rerun analysis.

报告层负责静态收集和解释，应该聚合上游输出，而不是重新运行分析。
