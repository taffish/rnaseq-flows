# TAFFISH RNA-seq Flows

Static documentation and example-report portal for the TAFFISH RNA-seq flow family.

[![Deploy GitHub Pages](https://github.com/taffish/rnaseq-flows/actions/workflows/pages.yml/badge.svg)](https://github.com/taffish/rnaseq-flows/actions/workflows/pages.yml)

This repository is intended to be published with GitHub Pages at:

- <https://taffish.github.io/rnaseq-flows/>
- Reference yeast report: <https://taffish.github.io/rnaseq-flows/examples/yeast-standard-report/>
- De novo yeast report: <https://taffish.github.io/rnaseq-flows/examples/yeast-denovo-standard-report/>
- Official executable `rnaseq-standard-flow` docs: <https://github.com/taffish/rnaseq-standard-flow/tree/main/docs>

## Purpose

`rnaseq-flows` is a user-facing project site, not a taf-flow package itself. It explains how the RNA-seq flow family fits together, links each subflow, provides input-preparation guidance, and hosts complete yeast SNF2 example reports generated by `taf-rnaseq-standard-flow` in reference and de novo modes.

The current public flow family uses `rnaseq-standard-flow 0.3.0-r1`, `rnaseq-report-flow 0.3.0-r2`, and `0.2.0-r1` reference/de novo subflows. The `0.3` standard flow preserves normal command-line defaults while exposing advanced `@step-name: ... @:` bridges to selected internal tool calls for expert tuning. The report flow renders standalone HTML reports whose primary plots, tables, and QC/report child pages are embedded as report payloads, so the main report can be shared and opened offline as a single HTML file.

For users who want to run their own data, the authoritative command-line
manuals live with the executable flow repository:
<https://github.com/taffish/rnaseq-standard-flow/tree/main/docs>.

The site keeps the tool logic and the public demonstration separate:

- taf-flow repositories contain executable flow code and tests.
- this repository contains static HTML documentation, a curated example report, and GitHub Pages assets.
- large raw sequencing data should remain in data apps or external archives, not in this static site.

## Contents

- `index.html`: project landing page and route overview.
- `flows/`: flow-family map and subflow roles.
- `docs/flows/`: per-flow usage manuals with inputs, parameters, outputs, and connection notes.
- `docs/quickstart.html`: standard-flow quick start and common run modes.
- `docs/input-preparation.html`: FASTQ, samples table, reference resources, gene-set inputs, and de novo annotation resources.
- `docs/data-sources.html`: public data-source, attribution, and provenance summary for the yeast examples.
- `docs/report-interpretation.html`: guide for reading the final HTML report.
- `docs/faq.html`: practical questions and boundary notes.
- `examples/yeast-standard-report/`: static reference-route yeast example report, embedded payloads, and supporting report artifacts.
- `examples/yeast-denovo-standard-report/`: static no-reference yeast example report, embedded payloads, and supporting report artifacts.
- `.github/`: issue templates, pull request template, and no-build Pages workflow.
- `scripts/check-links.js`: curated static link checker.

## Local Preview

The site is plain static HTML/CSS/JS. From this directory, run:

```sh
python3 -m http.server 8000
```

Then open:

```text
http://127.0.0.1:8000/
```

Opening `index.html` directly also works for most pages, but a local server is closer to GitHub Pages behavior.

## Validation

Run the curated local link checker before publishing or opening a pull request:

```sh
node scripts/check-links.js
```

The checker validates the hand-maintained site pages and skips generated
third-party report internals such as MultiQC and Qualimap HTML.

## Updating the Yeast Examples

1. Regenerate the reference and/or de novo yeast report with the current `taf-rnaseq-standard-flow`.
2. Prefer the `03_results/report/` package produced inside the completed standard-flow output; it contains the report-flow output tree used by this site.
3. Replace the matching static report directory:
   - `examples/yeast-standard-report/`
   - `examples/yeast-denovo-standard-report/`
4. Keep these entry points present for each example:
   - `examples/yeast-standard-report/index.html`
   - `examples/yeast-standard-report/04_reports/rnaseq_report.html`
   - `examples/yeast-standard-report/04_reports/report_interpretation.html`
   - `examples/yeast-denovo-standard-report/index.html`
   - `examples/yeast-denovo-standard-report/04_reports/rnaseq_report.html`
   - `examples/yeast-denovo-standard-report/04_reports/report_interpretation.html`
5. Update the hand-written overview metrics from `04_reports/key_metrics.tsv`
   and `04_reports/project_summary.tsv`.
6. Re-run a local static link check before publishing.

## Yeast Example Data Sources

The public yeast example reports are generated from the same source project:
ENA / NCBI BioProject [`PRJEB5348`](https://www.ebi.ac.uk/ena/browser/view/PRJEB5348),
a *Saccharomyces cerevisiae* WT versus `snf2` knockout RNA-seq experiment.
The example packages are prepared by
[`taffish/rnaseq-yeast-get-data`](https://github.com/taffish/rnaseq-yeast-get-data),
not by this static website repository.

The preparation app records selected biological replicates, lane accessions,
actual download source, fallback use, checksums, commands and manifests. It
also prepares author-derived count resources from
[`bartongroup/profDGE48`](https://github.com/bartongroup/profDGE48), SGD S288C
`R64.4.1` reference files, and SGD/GO-derived gene-set resources. The public
site summary is available at
<https://taffish.github.io/rnaseq-flows/docs/data-sources.html>.

When reusing the example data or reports, cite the original yeast RNA-seq study
and the relevant database/resource accessions, and follow the current terms of
ENA/EMBL-EBI, NCBI, SGD, Gene Ontology and the upstream author repository.

## GitHub Pages

Recommended repository name:

```text
rnaseq-flows
```

Two Pages modes are supported:

### GitHub Actions mode

Use the bundled workflow:

- Settings -> Pages -> Source: `GitHub Actions`
- Workflow: `.github/workflows/pages.yml`

The workflow runs `node scripts/check-links.js`, uploads the static repository
root, and deploys it to Pages.

### Branch root mode

If you prefer the simplest static-site setup:

- Source: `main`
- Directory: repository root
- Custom build: none

The `.nojekyll` file is included so GitHub Pages serves report assets and generated paths without Jekyll processing.
